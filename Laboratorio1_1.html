Concepts

URL

Son las siglas correspondientes a: "Uniform Resource Locator".

Se trata de la secuencia de caracteres que sigue un estándar y que permite denominar recursos dentro del entorno de Internet para que puedan ser localizados.

Referencia: Julián Pérez Porto y María Merino. (2010). Definición de URL. 14 de Octubre de 2018, de Definicion.de Sitio web: https://definicion.de/url/

MIME

Son las siglas correspondientes a: "Multipurpouse Internet Mail Extensions".

Permite mejorar la posibilidad de transferencia de texto entre distintos idiomas y alfabetos en el envío de emails, y también sirve para el envío de archivos a través del protocolo HTTP.

Questions

¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es la Red de redes, mientras que la WWW es un sistema de hipertextos accesibles mediante Internet.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

Son un conjunto de métodos de petición para indicar la acción que se desea realizar para un recurso determinado.
El método GET solicita una representación de un recurso específico. Las peticiones que usan el método GET sólo deben recuperar datos.
El método HEAD pide una respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta.
El método POST se utiliza para enviar una entidad a un recurso en específico, causando a menudo un cambio en el estado o efectos secundarios en el servidor.
El modo PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición.
El método DELETE borra un recurso en específico
El método PATCH es utilizado para aplicar modificaciones parciales a un recurso.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Los formularios están delimitados con la etiqueta FORM Esta etiqueta agrupa varios elementos del formulario, como botones y casillas de texto, y posee 2 atributos: METHOD y ACTION.
Se utiliza está etiqueta debido a los distintos atributos que se pueden utilizar de ella.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Se utiliza el método GET, para obtener la información del servidor

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Es la respuesta estándar para peticiones correctas. Y no, no es un error es un resultado de petición correcta.

Significa que la página no se encontro. El servidor realizó la petición, sin embargo no encontro el recurso solicitado

No, ya que el error es un error interno del servidor.

Ciclo de vida de sistemas de información

De acuerdo con Jean Carlos (2010) un sistema de información es un conjunto de elementos que trabajan entre sí con el fin de apoyar un proyecto o negocio. De igual manera dice que el ciclo de vida de un sistema de infromación gerencial debe incluir : Planificación, análisis, diseño, implementación y uso y mantenimiento.

Según Fernando Berzal (S.F.):
La planificación consiste en la determinación del proyecto y su viabilidad, análisis de riesgos y la asignación de recursos.
El análisis consiste en la comprension de las necesidades del cliente y el establecimiento de los requerimientos del sistema.
El diseño consiste en la representación de las caracterísiticas del sistema para realizar una implementación adecuada.
La implementación consiste en la selección de las herramientas adecuadas para la solución de las necesidades del cliente. En esta parte, es necesario que la programación lleve una estructura decifrable y todos los identificadores relacionados al funcionamiento de una variable.
El uso y mantenimiento segun Berzal (S.F.) es la etapa más importante del ciclo de vida, ya que mantiene al sistema en funcionamiento mediante 3 etapas: eliminación de defectos, adaptación de nuevas necesidades y adición de nuevas funcionalidades.

Ciclo de desarrollo de sistemas de información

Según Anónimo (S.F) es un proceso por el cual los analistas de sistemas, los ingenieros de software, los programadores y los usuarios finales, se relacionan y estudian la situación actual con el objetivo de elaborar un sistema de información
Las fases de desarrollo según Avelino Felipe Plicarpio (2021) son:
Identificación del problema, opoartunidades y objetivos: Esta primera fase requiere que el analista obseve objetivamente lo que sucede en un negocio, para así encontrar los problemas y determinar el área de oportunidad del sistema de información. En esta, también es necesario comprender el objetivo del cliente.
Determinacion de los requerimientos de informacion: En está segunda fase, el analista busca comprender las necesidades e información que el cliente necesita para llevar a cabo sus actividades.
Diseño del sistema: En esta fase, se producen los detalles que establecen de que manera el sistema cumplirá con los requerimientos de la fase anterior.
Desarrollo del software: Se desarrollan los programas necesarios para que el sistema funcione. En esta etapa es muy importante que los desarrolladores lleven la docuementación.
Prueba del sistema: Cada programa es probado. En esta fase, de manera experimental, se asegura de que el sistema no tiene fallas.
Implementación y evaluación: En está fase, se realiza la instalación y entrenamiento de los usuarios. La evalución es necesaria para identificar los puntos débiles del sistema.
